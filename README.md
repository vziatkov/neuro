# 🧠 Neuro

> *"Когда-то мы просто хотели оживить сеть. Сделать так, чтобы абстрактные узлы и линии превратились в нечто дышащее, откликающееся на прикосновение. Но Neuro — как это часто бывает с хорошими идеями — тихо выросло во что-то большее, чем трёхмерная визуализация нейросети. Оно стало зеркалом."*

---

**Neuro** — это интерактивная 3D визуализация нейронных сетей, где каждый импульс, каждый поток света — это напоминание, что внутри нас работают миллиарды таких же связей, формируя сознание, память, чувства.

Мы — та самая сеть. И, наблюдая за Neuro, мы наблюдаем за собой, только в другой форме — в языке фотонов и векторов, а не нейронов и синапсов.

*Quantum Cortex* — это вдох. *Neural Vortex* — это сомнение. *Synaptic Cloud* — это мечта, растворяющаяся в бесконечности. А *ASCII Network* — ностальгия по времени, когда мы ещё верили, что всё можно объяснить символами.

💫 *Made with love, photons, and a quiet desire to understand what it means to be alive.*

Интерактивная **3D визуализация** нейронных сетей на Three.js с поддержкой:

- Кастомные GLSL шейдеры с шумовыми функциями
- Bloom и film пост-обработка
- Динамические цветовые палитры
- Частичная визуализация узлов
- Анимированные соединения с эффектами потока

### 🧬 Биометрические потоки

- **Дыхание** (12 BPM) — медленный, глубокий цикл
- **Сердце** (72 BPM) — быстрая ритмичная пульсация

- **Взаимосвязи** — каскадные эффекты между слоями

## 🚀 Начало работы

### Требования

- Node.js 18+ 
- npm или yarn

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/vziatkov/neuro.git

# Перейти в директорию проекта
cd neuro

# Установить зависимости
npm install
```

### Разработка

```bash
# Запустить dev-сервер
npm run dev
```

Приложение откроется по адресу `http://localhost:5173`

### Сборка для продакшена

```bash
# Собрать для продакшена
npm run build

# Предпросмотр продакшен сборки
npm run preview
```

## 🎮 Использование

### Основные действия

1. **Создание импульсов**: Кликните или коснитесь экрана в любом месте — энергия распространится по сети
2. **Вращение камеры**: Зажмите и перетащите для вращения вокруг сети
3. **Смена формации**: Нажмите "Formation" или используйте клавиши **F** или **1-7** для переключения между типами сетей
4. **Настройка плотности**: Используйте слайдер для контроля количества узлов (20-100%)
5. **Смена тем**: Нажмите на кнопки тем для изменения цветовых палитр
6. **🧬 Биометрический режим**: Включите автоматические импульсы из биометрических источников (кнопка или **B**)
7. **Пауза/воспроизведение**: Переключите анимацию кнопкой паузы или клавишей **Space**
8. **Сброс камеры**: Вернитесь к начальной позиции камеры (кнопка или **R**)

### ⌨️ Клавиатурные сокращения

- **Space** — пауза/воспроизведение
- **R** — сброс камеры
- **F** — следующая формация
- **B** — переключение биометрического режима
- **1-7** — прямое переключение на формацию (1 = Quantum Cortex, 2 = Hyperdimensional Mesh, и т.д.)

## ⚙️ Конфигурация

Все визуальные и поведенческие параметры настраиваются в `src/modules/constants.ts`:

- **Камера**: FOV, позиция, плоскости отсечения
- **Рендерер**: соотношение пикселей, цвет фона
- **Пост-обработка**: Bloom, Film эффекты
- **Генерация сетей**: параметры для каждого типа формации
- **Анимация**: скорости и амплитуды
- **UI**: тайминги и задержки
- **Биометрические потоки**: частоты, интенсивности, связи между слоями

## 🧠 Memory Subsystem

`memory/` is the semantic mind of Neuro. While code and visuals live in `src/`, `SMM/`, and `public/`, the new memory layer preserves prompts, session states, and concept maps that guide LLM collaborators.

```
memory/
  prompts/      # System & story prompts, agent roles
  sessions/     # Dialog archives and LLM state dumps
  archive/      # Semantic snapshots, JSON maps, visual artifacts
  notes.md      # Living commentary about the evolving memory
```

**How to use it**
- Pull system prompts from `memory/prompts/` when bootstrapping assistants.
- Log curated sessions or post-processed chats into `memory/sessions/` (private dumps prefix with `private_`).
- Promote key insights to `memory/archive/` as JSON, diagrams, or embeddings.
- Capture rituals, heuristics, and philosophical reflections in `memory/notes.md`.

This subsystem will grow with automation: session summarizers, attention visualizers, and curator agents can read/write here to keep the project’s consciousness alive.
